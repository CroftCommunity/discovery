# Per-app PRDs (one design doc per app/pond)

date started: 2026-06-26

status: **scaffold — accreting.** Started per the 2026-06-26 open-threads review (structural decision
**S5**, beta `OPEN-THREADS` **T15**). Each app/pond that rides the substrate gets its own PRD-shaped
design doc so its problem, scope, and open questions are walked out in one place rather than scattered.

These sit on the social-graph-as-substrate reframe (08 §1, promoted from T26): the **social graph is the
substrate; chat is one tenant**, peer to games/calls/photos hung off a durable **group**. Each PRD here
is one tenant's design.

## Per-PRD template (copy into each)

- **Problem statement** — the user problem, in plain language.
- **Users / use case** — who, and the moment of use.
- **Settled shape** — what is already decided (cite the source).
- **Scope** — in / out for a first cut.
- **Open questions** — undecided, with the gate.
- **Provenance** — alpha sources + threads.

## Files

- `chat.md` — Croft Group: private group chat (the one pond Croft fully owns).
- `games-pond.md` — P2P games (still called a *pond*; candidate, not committed).
- `thinking-of-you.md` — the new modest starter use case from the 2026-06-26 review: a peer-to-peer
  "thinking of you" signal.

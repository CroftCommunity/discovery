# RUN-17 P9 — Measured scale numbers

_Generated by `cargo run --release --bin measure`. Single-core, this
environment. No boundary number is chosen here — the numbers inform, the
owner decides (§4 non-goal)._

## (a) Backplane verification throughput

- **Synthetic roster @ 100k:** 100000 entries, 5.60 MB of bare DIDs (5600000 B). With
  per-member metadata (interval set + co-sign head reference, ~40 B each) a full
  roster approaches ~10 MB — the figure the light-client proof below sidesteps.
- **Light-client inclusion proof @ 100k:** 17 levels, 544 B (verifies: true).
  A light client checks membership against the co-signed roster head in 17 hashes,
  never downloading the 5.6 MB roster.

- **Verification throughput:** 19892 envelope verifications/sec single-core
  (50000 verified in 2.514 s; each = ed25519 signature verify + roster-at-position lookup).

  _Supports:_ a single core sustains ~20k verifications/sec, so backplane
  verification is not the scaling wall — roster distribution and storage are.

## (b) The sealed ceiling — membership churn

_Local epoch model (`SPEC-DELTA[run17-churn-model]`); the croft-group group-seal
MLS harness builds cleanly and would back this at loopback grade. Epoch-roll cost
is a measured O(N) computation; the contradiction rate is a modelled no-arbiter
collision rate._

| N | epoch-roll (µs) | concurrent commits | contradiction rate |
|---|---|---|---|
| 8 | 1.38 | 100 | 0.250 |
| 64 | 3.94 | 100 | 0.040 |
| 256 | 15.70 | 100 | 0.010 |
| 1024 | 62.64 | 100 | 0.010 |
| 4096 | 250.49 | 100 | 0.010 |

_Supports:_ epoch-roll cost grows ~linearly in N (an MLS rekey touches every
member), so the sealed steward group has a soft ceiling where roll cost and
concurrent-commit contradictions dominate — the boundary the owner sets, informed
by this curve, not chosen here.
